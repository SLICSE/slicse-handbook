# GitHub õpikeskkonnana

See jaotis selgitab, kuidas kasutada GitHubi digiloovtöö õpikeskkonnana. SLICSE käsiraamatus ei käsitleta GitHubi ainult programmeerimisvahendina, vaid nähtava töövoo, dokumentatsiooni, tagasiside ja õpetaja orkestreerimise keskkonnana.

Digiloovtöö on pikemaajaline meeskonnapõhine arendus- ja õppeprotsess. Sellises protsessis vajab õpetaja ülevaadet sellest, mida meeskonnad teevad, millised ülesanded on pooleli, kus tekivad takistused, kuidas kasutatakse tagasisidet ja millised tõendusmaterjalid näitavad õppimise edenemist. GitHub aitab seda tööprotsessi nähtavaks teha.

## GitHubi roll digiloovtöös

GitHub võimaldab digiloovtöös siduda mitu olulist tööprotsessi osa:

* repository ehk projekti tööruum;
* Issues kui backlog ja tööülesannete kogum;
* GitHub Projects kui Kanbani töövoog;
* milestones kui sprintide või vahe-eesmärkide tähistajad;
* README ja Wiki kui dokumentatsiooni koht;
* comments kui tagasiside ja arutelu koht;
* commits ja muud tegevusjäljed kui õppeprotsessi tõendusmaterjalid.

GitHub ei asenda õpetaja juhendamist. See toetab õpetajat, muutes meeskondade tööprotsessi, ülesanded, otsused, dokumentatsiooni ja tagasiside nähtavamaks.

## Mida siit jaotisest leiab?

* selgituse, miks GitHub võib olla kasulik digiloovtöö juhendamisel;
* GitHubi põhimõisted õpetajale arusaadavas keeles;
* juhised repository kasutamiseks projekti tööruumina;
* Issues kasutamise backlog’i ja tööülesannetena;
* GitHub Projects kasutamise Kanbani töövoona;
* milestones kasutamise sprintide või vahe-eesmärkidena;
* README ja Wiki kasutamise dokumentatsioonina;
* õppija, meeskonna ja õpetaja töövoo kirjeldused GitHubis;
* tagasiside andmise võimalused GitHubis;
* õppija panuse ja tööprotsessi jälgimise põhimõtted;
* esimese GitHubi tunni korraldamise soovitused;
* kokkulepped ja riskikohad GitHubi kasutamisel.

## GitHub kui nähtav töövoog

Digiloovtöös on oluline, et tööprotsess ei jääks ainult õpilaste omavaheliseks suuliseks kokkuleppeks. GitHub võimaldab muuta tööprotsessi nähtavaks:

* mida on vaja teha;
* kes mille eest vastutab;
* mis on pooleli;
* mis vajab tagasisidet;
* mis on valmis;
* millised otsused on tehtud;
* kuidas prototüüp ja dokumentatsioon arenevad;
* milline on iga õppija panus meeskonna töösse.

Selline nähtavus toetab nii õppija enesejuhtimist, meeskonnatööd kui ka õpetaja orkestreerimist.

## Selle jaotise põhimõte

GitHubi kasutamise eesmärk digiloovtöös ei ole muuta kõiki õpilasi arendajateks. Eesmärk on toetada nähtavat, juhendatud ja tagasisidestatud tööprotsessi, kus õppijad õpivad kavandama, ülesandeid jaotama, koostööd tegema, dokumenteerima, tagasisidet kasutama ja oma panust reflekteerima.
